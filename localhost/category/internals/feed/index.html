They are currently the easiest way to serve Ubuntu packages to users through a repository that will also push out updates when they are available. Sadly, it is not as easy as it could have been, but it works, and for the end user it is very simple.

That’s why we set up our own PPA for the Computational Physics group. A PPA is something which works best if several people are using and contributing to it. If you want to help out with maintaining the Computational Physics PPA, you need to do the following:

  1. Create a user on Launchpad.
  2. Create a GPG key if you don’t have one: 
    gpg --gen-key
  3. Find the ID and fingerprint of your key: 
    gpg --list-keys

    The ID should be a string of about 8 letters and digits, for instance 9ABC6F7G.
    The fingerprint is on the form 27E0 7815 B47C 0397 90D5  8589 27D9 A27B F3F9 6058

  4. Upload this to the Ubuntu key server: 
    gpg --send-keys --keyserver keyserver.ubuntu.com YOUR_KEY_ID
  5. Log in to your Launchpad account, visit your profile and click the yellow pencil icon next to OpenPGP keys. Insert your key fingerprint and click Import Key.

You are now registered and ready to help out maintaining. Send me a message and I will give you access to the Computational Physics team.

You may for instance help out with the packages, which I will write a guide on later, and to have a look at what’s currently available you may either add the PPA and download a source package:

sudo apt-add-repository ppa:comp-phys/stable
sudo apt-get update
apt-get source comp-phys

This may then be reuploaded later using debuild and dput:

cd comp-phys
( do some changes )
debuild -S
dput ppa:comp-phys/stable ../comp-phys-minimal_0.0.2_source.changes

Or you could clone the git repository that also contains the package, which should be located at our GitHub account. This is currently where the comp-phys and comp-phys-minimal package sources reside, which you find in this repository.
